## v5.1.0 (2026-05-08)

**安全加固版本：修复高优先级和中优先级差距**

### 后端安全加固

| 功能 | 实现方式 | 状态 |
|------|----------|------|
| **CORS 白名单** | 配置化白名单，默认禁用 `*` | ✅ |
| **请求限流** | slowapi 中间件，按接口类型限流 | ✅ |
| **Refresh Token** | 双 Token 机制，30天有效期 | ✅ |
| **数据库连接池** | MySQL 连接池配置（pool_size=10） | ✅ |
| **输入长度限制** | Pydantic 字段加 `max_length` | ✅ |
| **日志中间件** | 请求/响应日志记录 | ✅ |
| **全局异常处理** | 统一异常响应，生产环境隐藏详情 | ✅ |

### 后端配置文件更新

- `app/config.py` - 添加：
  - `ENV_MODE` 环境模式
  - `CORS_ORIGINS` 白名单配置
  - `REFRESH_TOKEN_EXPIRE_DAYS` / `REFRESH_TOKEN_SECRET_KEY`
  - `RATE_LIMIT_PER_MINUTE` 等限流配置
  - `DB_POOL_*` 连接池配置

- `app/database.py` - 优化：
  - 支持 MySQL 连接池
  - SQLite WAL 模式优化
  - 添加 `check_db_connection()` 健康检查

- `app/schemas.py` - 增强：
  - 所有字段添加 `max_length` 限制
  - 数值字段添加 `ge/le` 范围校验
  - 手机号格式正则验证
  - 密码强度检查
  - 添加 `RefreshTokenRequest` schema

- `app/auth.py` - 增强：
  - `create_refresh_token()` 新函数
  - `verify_refresh_token()` 验证函数
  - Token 类型区分（access/refresh）
  - 向后兼容的 `verify_token()`

- `app/routers/auth.py` - 增强：
  - 注册/登录添加限流注解
  - 新增 `/api/auth/refresh` 端点
  - 返回 Token 包含 `refresh_token`
  - 详细错误信息

- `app/main.py` - 重构：
  - 引入 slowapi 限流器
  - 添加请求日志中间件
  - CORS 配置使用白名单
  - 统一异常处理
  - 健康检查添加数据库连接状态

## v5.0.0 (2026-05-04)

**重大更新：完整修复10项业务功能缺失**

### 后端核心功能修复

| 功能 | 实现 | 说明 |
|------|------|------|
| **订单超时自动取消** | APScheduler定时任务 | 每5分钟扫描，30分钟未支付自动取消 |
| **库存扣减+并发锁** | SELECT FOR UPDATE行级锁 | 下单扣库存，取消/退款回滚，防止超卖 |
| **审计日志** | AuditLog模型+中间件 | 记录所有关键操作，支持按用户/操作/日期筛选 |
| **库存预警API** | inventory路由 | 低库存查询、库存汇总、批量更新 |
| **Excel导出** | openpyxl | 订单导出(13字段)、菜品销售报表，含格式化表头 |
| **财务报表** | finance路由 | 日报/月报/年报，含支付方式/订单类型/趋势分析 |
| **云打印框架** | CloudPrinter类 | 对接易联云/飞鹅云打印，订单小票+后厨联 |

### 前端功能修复

| 功能 | 实现 | 说明 |
|------|------|------|
| **扫码自动绑定桌台** | ScanTable页面 | URL参数(store_id+table_id)自动绑定 |
| **菜单库存预警** | Menu.tsx | 低库存显示'仅剩X份'，售罄禁用加购 |
| **骑手地图导航** | Orders.tsx | 腾讯地图URL调起，一键导航 |
| **库存管理Dashboard** | Inventory.tsx | 商家端3级库存状态，+-调库存 |

## v4.1.0 (2026-05-04)

**更新：完整微信支付模块**

### 后端微信支付核心 (WxPay)
- **签名工具**：支持 MD5 和 HMAC-SHA256 签名算法
- **统一下单**：对接微信 `pay/unifiedorder`，支持 JSAPI/H5 支付
- **回调验签**：自动验证微信支付回调签名，防止伪造
- **订单查询**：`pay/orderquery` 主动查询支付状态
- **退款**：`secapi/pay/refund` 支持部分/全额退款（需商户证书）
- **关单**：`pay/closeorder` 关闭未支付订单
- **XML工具**：字典↔XML互转，适配微信协议

## 版本命名规则

- `v{主版本}.{次版本}.{修订}`
- 主版本：重大架构变更或商业模式调整
- 次版本：新增功能模块
- 修订：Bug修复、体验优化
